Was ist Informatik? 🧠

Informatik ist die "Wissenschaft von der systematischen Darstellung, Speicherung, Verarbeitung und Übertragung von Informationen". Der Begriff selbst ist eine Kombination aus Information und Automatik. Sie ist weit mehr als nur "Programmieren" und gliedert sich in vier Hauptbereiche:

  • Theoretische Informatik: Die mathematischen und logischen Grundlagen. Was kann überhaupt berechnet werden? (z.B. Automaten- und Komplexitätstheorie).
  • Praktische Informatik: Die Entwicklung von Software und die Lösung konkreter Probleme (z.B. Algorithmen, Datenstrukturen, Betriebssysteme, Compilerbau).
  • Technische Informatik: Die Brücke zur Elektrotechnik. Wie baut man einen Computer? (z.B. Rechnerarchitektur, Hardware-Design).
  • Angewandte Informatik: Der Einsatz von Informatik in anderen Fachbereichen (z.B. Wirtschafts-, Medien-, Medizininformatik).

Die Von-Neumann-Architektur 🏛️

Fast jeder Computer, den du heute benutzt, folgt diesem grundlegenden Bauplan von John von Neumann aus dem Jahr 1946. Die Architektur besteht aus vier Hauptkomponenten:

  • Rechenwerk (ALU): Führt arithmetische (z.B. +, -) und logische (z.B. AND, OR) Operationen durch.
  • Steuerwerk: Interpretiert die Befehle eines Programms und steuert das Rechenwerk und den Speicher. Rechen- und Steuerwerk bilden zusammen den Prozessor (CPU).
  • Speicherwerk (Hauptspeicher): Speichert sowohl die Anweisungen (das Programm) als auch die Daten, die verarbeitet werden. Dies ist das entscheidende Kernprinzip der Von-Neumann-Architektur!
  • Ein-/Ausgabewerk (I/O): Ermöglicht die Kommunikation mit der Außenwelt (z.B. Tastatur, Bildschirm).

Diese Komponenten sind durch ein Bus-System verbunden, das Daten und Adressen zwischen ihnen transportiert.

Bits, Bytes & Datendarstellung

Die kleinste Informationseinheit im Computer ist das Bit, das zwei Zustände annehmen kann: 0 oder 1. Um komplexere Informationen darzustellen, werden mehrere Bits zu Folgen kombiniert. Mit N Bits kann man \(2^N\) verschiedene Zustände darstellen.

Die kleinste adressierbare Speichereinheit ist in der Regel ein Byte, das aus 8 Bits besteht und somit \(2^8 = 256\) verschiedene Werte annehmen kann.

Daten sind die reine Repräsentation von Information in Form von Bits. Die Interpretation dieser Daten (z.B. als Zahl, Buchstabe oder Pixel) verleiht ihnen erst ihre Bedeutung.

Zahlensysteme im Detail 🧮

Alle Zahlensysteme basieren auf dem Stellenwertprinzip: Der Wert einer Ziffer hängt von ihrer Position ab. Der Wert einer Zahl ergibt sich aus der Summe der Ziffern, multipliziert mit dem jeweiligen Stellenwert (einer Potenz der Basis).

Umrechnung → ins Dezimalsystem

Multipliziere jede Ziffer mit dem Wert ihrer Stelle (Basis hoch Position) und addiere die Ergebnisse.

Binär → Dezimal: \(101011_2\)
= 1·\(2^5\) + 0·\(2^4\) + 1·\(2^3\) + 0·\(2^2\) + 1·\(2^1\) + 1·\(2^0\)
= 32 + 0 + 8 + 0 + 2 + 1 = 43₁₀

Hexadezimal → Dezimal: \(1A7_{16}\) (wobei A=10)
= 1·\(16^2\) + 10·\(16^1\) + 7·\(16^0\)
= 256 + 160 + 7 = 423₁₀

Umrechnung: Dezimal → Binär (Ganze Zahlen)

Dieser Algorithmus nutzt die Division mit Rest.

  1. Teile die Dezimalzahl durch 2.
  2. Notiere den Rest (0 oder 1). Dies ist die nächste Ziffer deiner Binärzahl (von rechts nach links).
  3. Ersetze die Zahl durch das ganzzahlige Ergebnis der Division.
  4. Wiederhole die Schritte, bis das Ergebnis 0 ist.

Beispiel für 43:
43 / 2 = 21 Rest 1
21 / 2 = 10 Rest 1
10 / 2 = 5 Rest 0
5 / 2 = 2 Rest 1
2 / 2 = 1 Rest 0
1 / 2 = 0 Rest 1
Ergebnis (von unten nach oben gelesen): 101011₂

Umrechnung: Dezimal → Binär (Nachkommastellen)

Hier wird der Nachkommaanteil wiederholt mit 2 multipliziert.

  1. Multipliziere den Nachkommaanteil mit 2.
  2. Die Vorkommastelle des Ergebnisses (0 oder 1) ist die nächste Ziffer deiner Binärzahl (von links nach rechts).
  3. Ersetze den Wert durch den neuen Nachkommaanteil.
  4. Wiederhole, bis der Nachkommaanteil 0 ist (oder die gewünschte Genauigkeit erreicht ist).

Beispiel für 0,6875:
0.6875 * 2 = 1.375
0.375 * 2 = 0.75
0.75 * 2 = 1.5
0.5 * 2 = 1.0
Ergebnis (von oben nach unten gelesen): 0,1011₂

Von der Idee zum Programm 💡

Die Erstellung von Software ist ein mehrstufiger Prozess, der Abstraktion erfordert. Man beginnt mit einem Problem aus der realen Welt und übersetzt es schrittweise in eine Form, die ein Computer versteht.

  1. Problem: Eine umgangssprachliche, oft mehrdeutige Beschreibung.
  2. Anforderungsspezifikation: Eine präzisere Beschreibung dessen, was das Programm leisten soll.
  3. Modell und Algorithmen: Eine formale, aber noch nicht vollständig detaillierte Beschreibung der Lösungslogik.
  4. Programmtext (Quellcode): Die Implementierung des Algorithmus in einer Programmiersprache mit exakter Syntax. Dies ist die Aufgabe des Programmierers.
  5. Maschinenprogramm (Zielprogramm): Der vom Computer direkt ausführbare Code. Dies ist die Aufgabe von Compiler/Interpreter.

Compiler vs. Interpreter

Um Quellcode in Maschinencode umzuwandeln, gibt es zwei grundlegende Ansätze:

Compiler (z.B. für C++)

Ein Compiler übersetzt das gesamte Programm vor der Ausführung in eine ausführbare Datei.
+ Vorteil: Sehr schnelle Ausführung.
- Nachteil: Das Ergebnis ist plattformabhängig (z.B. nur für Windows x64). Für jede Plattform wird ein eigener Compiler benötigt.

Interpreter (z.B. für Python)

Ein Interpreter liest den Quellcode Zeile für Zeile und führt die Anweisungen direkt aus.
+ Vorteil: Plattformunabhängig (benötigt nur den Interpreter für die Zielplattform).
- Nachteil: Langsamere Ausführung als kompilierter Code.

Java's Ansatz: Java kombiniert beide Welten. Der Java-Compiler (`javac`) erzeugt einen plattformunabhängigen Bytecode. Dieser wird von der Java Virtual Machine (JVM) ausgeführt (`java`), die den Bytecode interpretiert oder mit einem Just-in-Time (JIT) Compiler zur Laufzeit in schnellen, nativen Maschinencode übersetzt. Dies vereint die Plattformunabhängigkeit mit guter Performance.

Die Bausteine eines Java-Programms: Token

Jedes Programm lässt sich in seine kleinsten sinnvollen Einheiten, die Tokens, zerlegen. Whitespace (Leerzeichen, Tabs, Zeilenumbrüche) dient nur der Trennung von Tokens und wird vom Compiler ignoriert.

Struktur & Konventionen

Eine Java-Quellcodedatei (`.java`) besteht typischerweise aus:

  • Einer optionalen `package`-Angabe.
  • Keiner, einer oder mehreren `import`-Anweisungen.
  • Einer oder mehreren Klassen- oder Interface-Definitionen. Pro Datei darf es maximal eine `public` Klasse geben.

Jedes ausführbare Java-Programm benötigt eine `main`-Methode als Einstiegspunkt. Sie muss exakt so aussehen (nur der Name `args` kann variieren):

public static void main(String[] args) { ... }

Zusätzlich zu den festen Regeln der Syntax gibt es wichtige Konventionen, die die Lesbarkeit erhöhen:

  • Klassennamen beginnen mit einem Großbuchstaben (z.B. `Rechner`).
  • Variablen- und Methodennamen beginnen mit einem Kleinbuchstaben (z.B. `meineZahl`, `berechneSumme()`).
  • Codeblöcke innerhalb von `{...}` werden eingerückt.

Variablen, Deklaration & Speicher

Eine Variable ist ein Name für einen Speicherbereich im Computer, in dem ein Wert abgelegt werden kann. Jede Variable in Java hat:

  • Einen Namen (Bezeichner), über den sie angesprochen wird.
  • Einen Typ, der festlegt, welche Art von Daten sie speichern kann (z.B. `int` für ganze Zahlen, `String` für Text) und wie viel Speicherplatz sie belegt.
  • Einen Wert, der im Speicher abgelegt ist.
  • Einen Speicherort.

Bevor eine Variable genutzt werden kann, muss sie deklariert werden. Dabei wird ihr Typ und Name festgelegt. Optional kann sie direkt initialisiert, also mit einem Startwert versehen werden. 


// Nur Deklaration: Der Variable 'ergebnis' wird Speicherplatz reserviert.
// Ihr Wert ist zunächst undefiniert.
int ergebnis;

// Deklaration und Initialisierung in einem Schritt.
// Die Variable 'zahl' wird erstellt und bekommt sofort den Wert 10.
int zahl = 10;

// Eine neue Zuweisung überschreibt den alten Wert.
zahl = 20;

Elementare Ein- und Ausgabe ⌨️

Um mit einem Programm interagieren zu können, benötigen wir Wege, Daten auszugeben und vom Benutzer einzulesen.

Interaktives Beispiel

Dieses Programm demonstriert die grundlegende Ein- und Ausgabe auf der Konsole.


// Importiert die Scanner-Klasse, die wir zum Lesen brauchen.
import java.util.Scanner;

public class Begruessung {
  public static void main(String[] args) {
    // 1. Ein Scanner-Objekt erzeugen, das von der
    //    Standard-Eingabe (System.in) liest.
    Scanner sc = new Scanner(System.in);

    // 2. Eine Aufforderung auf der Konsole ausgeben.
    //    System.out.println() macht einen Zeilenumbruch danach.
    System.out.println("Bitte gib deinen Namen ein:");

    // 3. Die next()-Methode liest die nächste Zeichenkette
    //    bis zum nächsten Leerzeichen ein und speichert sie.
    String name = sc.next();
    
    // 4. Den eingelesenen Namen in einer Begrüßung ausgeben.
    System.out.println("Hallo, " + name + "!");
  }
}
